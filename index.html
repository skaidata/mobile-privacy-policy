Настоящая Политика конфиденциальности описывает, какие данные могут обрабатываться при использовании мобильного приложения SKAI Data (далее — «Приложение»), а также цели, правовые основания и меры защиты таких данных.

Используя Приложение, Пользователь подтверждает, что ознакомился с настоящей Политикой и принимает ее условия.

1. Какие данные мы можем обрабатывать

При использовании Приложения могут обрабатываться следующие категории данных:

Данные учетной записи

  • адрес электронной почты
  • пароль (вводится Пользователем)
  • токены авторизации (access token, refresh token)

Данные профиля пользователя

  • идентификатор пользователя
  • имя и фамилия
  • должность
  • роль пользователя в системе
  • компания или организация
  • иные данные профиля, получаемые из серверных сервисов

Данные взаимодействия с AI-функциями

При использовании AI-функций приложения могут обрабатываться:

  • текстовые сообщения в AI-чат
  • голосовые сообщения
  • технические параметры запросов

Технические данные

  • настройки приложения (например тема интерфейса)
  • параметры соединения
  • служебные данные запросов API

2. Источники получения данных

Данные могут быть получены:

  • непосредственно от Пользователя (ввод данных, текстовые и голосовые сообщения);
  • автоматически при использовании Приложения;
  • из серверных систем и API, используемых Приложением.

3. Цели обработки данных

Персональные и технические данные обрабатываются исключительно для следующих целей:

  • аутентификация и авторизация Пользователя
  • предоставление доступа к функциям Приложения
  • отображение и обновление данных профиля
  • работа AI-функций (чат, распознавание речи, обработка голосовых сообщений)
  • обеспечение стабильной работы приложения
  • диагностика ошибок и повышение качества сервиса
  • соблюдение требований законодательства Республики Казахстан

4. Правовые основания обработки

Обработка данных осуществляется на следующих основаниях:

  • исполнение пользовательского соглашения и предоставление сервиса
  • согласие Пользователя (например, при доступе к микрофону)
  • выполнение требований законодательства Республики Казахстан
  • законный интерес Оператора в обеспечении безопасности и работоспособности системы

5. Разрешения устройства

Приложение может запрашивать следующие разрешения:

Доступ к микрофону
Используется для записи голосовых сообщений и работы голосовых функций.

Доступ к распознаванию речи
Используется для корректной работы голосовых команд и AI-функций.

Доступ в интернет
Необходим для обмена данными с серверными сервисами и API.

6. Передача данных третьим лицам

Данные могут передаваться:

  • серверным сервисам и API, обеспечивающим работу Приложения
  • сервисам обработки AI-запросов и голосовых сообщений
  • веб-ресурсам, открываемым внутри встроенного браузера приложения

Оператор не продает персональные данные пользователей третьим лицам.

7. Хранение данных

В процессе работы Приложения данные могут храниться:

  • на серверах backend-инфраструктуры
  • локально на устройстве пользователя (например настройки интерфейса)
  • во временных файлах устройства для обработки голосовых или текстовых запросов

Сроки хранения данных определяются внутренними регламентами компании и требованиями законодательства.

8. Безопасность данных

Для защиты информации применяются организационные и технические меры безопасности, включая:

  • использование защищенных каналов передачи данных
  • контроль доступа к системам
  • хранение токенов авторизации в защищенных хранилищах
  • мониторинг работы сервисов

  • отозвать согласие на обработку данных
